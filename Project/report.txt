3. Architecture (20 points)
Describe the overall architecture of this project, including but not limited to the programming language, framework used, the major components and how they are connected. Draw a diagram similar to the one below.

The project follows a full-stack client-server architecture. The frontend is built with React and communicates with the backend via a REST API. The backend is a Node.js server using Express, which acts as the bridge between the user interface and the MySQL database. Authentication is handled using JSON Web Tokens (JWT) to keep user sessions secure.

5. Implementation (40 points)
Describe how each specification is implemented. Specify and highlight the program file in which each specification is implemented.

Home Page and User Dashboard:
The website features a clean landing page with a hero section displaying the project title "CS418 Project". The user dashboard serves as a central hub for students to manage their profiles, displaying user information and allowing for security settings like 2FA to be toggled. The Home page is implemented in `Home.jsx` and the dashboard logic is in `Dashboard.jsx`.
Files: Project/client/src/Home.jsx, Project/client/src/Dashboard.jsx

Secure Login and 2FA:
The login flow is a two-step process. First, the user enters their email and password. If correct and 2FA is enabled, the server generates a 6-digit OTP and sends it via email. The user must then enter this code correctly to receive a JWT token for authentication. This logic is spread across the login route and the OTP verification route.
Files: Project/server/route/user.js, Project/client/src/Login.jsx, Project/client/src/VerifyOtp.jsx

Email Verification:
When a new user signs up, the system automatically sends a verification email using Nodemailer. The user cannot log in until they verify their email with the code provided. The registration and verification logic handles the creation of the user and the temporary OTP entry.
Files: Project/server/route/user.js, Project/client/src/Signup.jsx, Project/client/src/VerifyEmail.jsx

Password Management:
Users can change their passwords from the dashboard or initiate a reset if they forget them. The backend uses bcrypt to hash new passwords before storing them in the database, ensuring that plain-text passwords are never saved.
Files: Project/server/route/user.js, Project/client/src/ForgotPassword.jsx, Project/client/src/Dashboard.jsx
